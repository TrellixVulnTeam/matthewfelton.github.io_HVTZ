  • The fetch specification differs from jQuery.ajax() in the following significant ways: The Promise returned from fetch() won’t reject on HTTP error status even if the response
    is an HTTP 404 or 500. Instead, as soon as the server responds with headers, the Promise
    will resolve normally (with the ok property of the response set to false if the response
    isn’t in the range 200–299), and it will only reject on network failure or if anything
    prevented the request from completing.
